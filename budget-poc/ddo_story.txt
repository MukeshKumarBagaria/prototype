User Story:
	•	As a DDO Creator, I want to submit the budget estimation for Expenditure for Verification.
	•	As a DDO Verifier, I want to review, edit and forward the budget estimation for approval.
	•	As a DDO Approver, I want to review, edit and approve the budget estimation.
Pre-Conditions
	•	Budget Line Items to be mapped with DDO.
	•	Department-wise ceilings defined and approved by FD.
	•	Budget module activated by Finance Department.
	•	Budget Submission date within the time limit as per Budget Circular.
Process Flow
	•	Data Synchronization:
Based on Employee login, DDO linked with department details are fetched from HRMS through secure API.
	•	Auto-Fetch Components: System retrieves all budget line items assigned by BCO to DDO.
System retrieves actual expenditure for budget line items occurred during previous 5 years.
System retrieves the Budget Allocation of previous year.
	•	DDO Creator: For expenditure DDO Creator selects the budget line item and enters the budget estimation for next 3 years and revised estimation for current year.
DDO Creator enters the data for each budget line item and saves.
DDO Creator submits to verifier once all budget line data is ready.
	•	Rule Engine Validation:
Budget Estimate submitted should not be more that overall ceiling limit for expenditure. If limit is exceeded display error message
	•	DDO Verifier:
DDO Verifier views the budget estimation submitted by DDO creator. DDO Verifier can either edit or return to DDO Creator or forward the file to DDO Approver.
	•	DDO Approver:
DDO Approver views the budget estimation forwarded by DDO verifier. He can either edit or return to DDO Verifier or approve the file. The approved budget will be sent to BCO Creator.
	•	Audit Trail and Reporting: Each action creation, modification, verification, approval, rejection logged for audit.
